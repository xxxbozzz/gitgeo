# gitgeo 前端重写规范

## 目标

将 gitgeo 从"开发者监控台"升级为"GEO 运营管理系统"。审美对标 **Linear + Vercel + Stripe**——干净、专业、有呼吸感，不是暗黑黑客风。

## 技术选型

基于 **Vben Admin v5**（25K stars, Vue 3 + TypeScript + Tailwind + 暗亮双主题）起手，不要从零写。

```
框架:   Vben Admin v5 (或 Naive UI Admin)
语言:   TypeScript strict
组件:   框架自带的表格/表单/卡片/弹窗/侧边栏
图表:   ECharts 6 (复用现有)
API:    复用现有 axios + TanStack Query 层 (src/api/)
路由:   保持 /console/ 前缀
```

## 设计原则

1. **亮色默认，暗色可选**——不是暗黑毛玻璃。白底+精致边框+微阴影，给人专业工具的信任感
2. **留白充足**——每页内容区 max-width 1280px，不要撑满屏幕
3. **信息层级清晰**——标题→KPI 卡片→筛选区→表格，从上到下自然流动
4. **交互反馈细腻**——hover 微上浮、按钮有 loading、操作有 toast 提示
5. **中文优先**——所有文案简体中文，技术术语保持英文

## 页面清单（12 页）

### 1. 仪表盘 `/dashboard`
- 顶部：问候语 + 日期 + 系统健康点（绿/黄/红）
- 4 个 KPI 卡片：总文章、本月新增、平均质量分、AI 可见率
- 左：7 日产出趋势（ECharts 面积图，浅蓝色渐变）
- 右：待处理关键词队列（紧凑表格，前 5 条）
- 底：最新文章列表 + 最近运行记录，各占一半

### 2. 素材库 `/materials` ★ 核心管理页
- 5 个 Tab：关键词库 | 标题库 | 图片库 | 作者库 | 知识库
- 每个 Tab：顶部搜索+筛选栏 + 操作按钮（导入/新建）+ 表格 + 分页
- 表格操作列：编辑 / 删除（弹窗确认）
- 新建/编辑用右侧 Drawer 滑出表单（不是弹窗）
- 知识库 Tab 额外显示：文档名、切片数、向量化状态、上传时间

### 3. 提示词库 `/prompts` ★ 核心管理页
- 卡片网格（3 列），每个卡片：类型标签、名称、描述摘要、启用/停用开关
- 顶部：分类筛选 + 搜索 + 新建按钮
- 点击卡片 → Drawer：完整 Prompt 内容（代码块）、关联模型、使用次数、历史版本列表
- 新建/编辑用全屏 Drawer，内嵌 Markdown 编辑器

### 4. 模型配置 `/models`
- 列表卡片：模型名、提供商、类型(chat/embedding)、端点 URL
- 每行右侧：主模型/备用 标签 + 启用开关 + 测试连接按钮 + 编辑
- 添加模型：Drawer 表单（名称/提供商/端点/API Key/温度/Max Tokens）
- 测试连接：发送 ping 请求，显示延迟和状态

### 5. 任务调度 `/tasks`
- 卡片列表：任务名、调度规则(cron/持续/间隔)、日产量上限、状态
- 操作：立即运行 / 暂停 / 编辑 / 查看运行历史
- 新建任务：Drawer 表单（名称、调度类型、时间、日产量、关联关键词库）
- 运行历史：时间线组件，显示每次运行的起止时间、耗时、状态

### 6. 文章管理 `/articles`
- 表格：标题（可点击展开预览）、质量分（颜色条）、状态（Tag）、关键词、更新时间
- 筛选：状态下拉 + 最低分输入 + 关键词搜索
- 批量操作：全选 → 批量发布 / 批量导出
- 单篇操作：预览(Drawer 渲染 Markdown)、返修、回收、发布
- 发布按钮 → 弹窗：勾选目标平台 + 发布模式(草稿/直接发布)

### 7. 发布中心 `/publications`
- 表格：文章标题、平台(图标+名称)、模式、状态(Tag)、结果链接、时间
- 筛选：平台下拉 + 状态下拉 + 搜索
- 失败行高亮红色，有重试按钮
- 点击行 → Drawer：请求/响应 JSON、错误信息、重试历史

### 8. 能力库 `/capabilities`
- 表格：能力名称、分组、状态(活跃/禁用)、声明级别、置信度(星级)、证据源数
- 筛选：分组下拉 + 状态 + 搜索
- 点击 → Drawer：保守值 vs 高级值对照、条件说明、关联证据源列表
- 禁用操作需要二次确认

### 9. 关键词中心 `/keywords`
- 双栏布局：左（关键词表格 + 分页）、右（GEO 真空词队列 + 聚类分布）
- 表格列：关键词、搜索量、难度(进度条)、状态、绑定文章
- 真空词队列：紧凑列表，每行有"生成文章"快捷按钮

### 10. 运行记录 `/runs`
- 表格：运行 ID、触发方式、关键词、状态、当前步骤、开始时间、耗时
- 筛选：状态 + 触发方式 + 关键词搜索
- 点击 → Drawer：运行详情 + 步骤时间线（垂直 Timeline 组件）
- 自动刷新：每 20 秒

### 11. 知识图谱 `/graph`
- 全宽 ECharts 力导向图，最小高度 480px
- 节点颜色区分类型：文章(蓝)、关键词(绿)、能力(橙)
- 右侧图例面板
- 点击节点 → 高亮关联 + 侧边信息卡

### 12. 系统状态 `/system`
- 4 个状态卡：环境、数据库、LLM API、ChromaDB
- 构建信息：版本号 + 构建时间
- 资源用量：CPU/内存/磁盘（若有）

## 全局组件

| 组件 | 用途 |
|------|------|
| `StatCard` | KPI 卡片，大数值 + 标签 + 趋势箭头 |
| `StatusTag` | 状态标签，绿/黄/红/蓝，带圆点 |
| `DetailDrawer` | 右侧滑出详情面板，宽度 640-960px |
| `ConfirmDialog` | 删除/禁用确认弹窗 |
| `PageHeader` | 页面标题 + 描述 + 操作按钮区 |

## 与后端 API 对接

全部复用现有 API 层（`src/api/`），不新增端点。新管理页面（素材库/提示词/模型/任务）暂时使用前端 mock 数据，等后端补充 API 后再切换。

## 交付标准

1. `npm run dev` 启动，12 页全部可访问、可交互
2. 亮暗双主题可切换，暗色为次要主题
3. 移动端不要求适配（管理后台）
4. 8 张真实截图（1440×900），替换 `docs/images/screenshots/`
