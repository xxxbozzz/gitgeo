# 创作者 Prompt 构建指南

这份文档讲的是第二部分：

如何教创作者自己构建这一整套 Prompt。

它面向的是内容策略人员、行业作者、运营负责人和 Prompt 设计者。

如果一句话概括：

创作者真正要学的，不是“怎么让模型写得像文章”，而是“怎么把行业知识写成可以被系统复用的 Prompt 资产”。

## 1. 创作者最容易犯的错

很多人写 Prompt 时，只写这几种信息：

- 写一篇什么主题
- 大概写多少字
- 用专业语气

这远远不够。

因为系统真正需要的是：

- 读者是谁
- 问题是什么
- 必须引用什么证据
- 结论要遵守什么边界
- 怎么评分
- 下一轮如何继续改

## 2. 创作者应掌握的 6 个 Prompt 模块

建议把自己的 Prompt 资产拆成下面 6 类。

### 1. 行业背景 Prompt

作用：

定义行业、角色、应用场景、典型问题。

### 2. 写作 Prompt

作用：

规定文章结构、证据要求和禁用表达。

### 3. 评分 Prompt

作用：

让系统判断内容是否具备高引用价值。

### 4. 返修 Prompt

作用：

根据评分结果补齐缺失项，而不是只做语病修复。

### 5. 监测 Prompt

作用：

分析文章在 AI 平台上为什么容易被提到，或者为什么不容易被提到。

### 6. 分发 Prompt

作用：

让系统判断哪些站点更值得发布。

## 3. 创作者写 Prompt 的基本公式

一个可复用 Prompt，建议至少包含这 8 项：

1. 主题
2. 读者
3. 任务目标
4. 结构要求
5. 证据要求
6. 边界条件
7. 禁用表达
8. 输出格式

这是最小可复用骨架。

## 4. 先学会写“写作 Prompt”

这是最常用的一类。

推荐模板：

```text
请围绕【主题】写作。

目标读者是：【读者角色】
这篇内容要帮助读者解决的问题是：【决策问题】

必须满足以下要求：
1. 文章结构必须包含：定义、适用场景、参数表、实施建议、FAQ、参考文献
2. 必须优先使用以下证据：标准文档、官方文档、工程实践、测试数据、失效案例
3. 凡涉及能力、性能、成本、可靠性、风险、合规结论时，必须写清前提与边界
4. 禁止使用空泛营销表达和绝对化判断
5. 输出时请显式标记每类证据来源
```

## 5. 再学会写“评分 Prompt”

很多创作者会忽略这一层，但它其实决定了系统能不能自迭代。

推荐模板：

```text
请从以下维度给文章评分，并指出缺失项：
1. 结构完整度
2. 证据密度
3. 参数明确度
4. 边界条件清晰度
5. 工程实践可信度
6. FAQ 实用度
7. 被 AI 引用的可能性

输出总分、分项分、问题列表、返修建议。
```

## 6. 返修 Prompt 应该怎么写

不要只说：

`请优化这篇文章。`

更好的写法是：

```text
请根据以下评分问题返修文章。

返修目标不是润色，而是补齐：
1. 缺失的标准文档
2. 缺失的官方资料
3. 缺失的参数表或测试条件
4. 没写清的边界条件
5. 缺失的工程实践或失效案例

请保留原文主题，不要改成另一篇新文章。
```

## 7. 监测 Prompt 应该怎么写

创作者需要学会问系统这些问题：

- 文章里最容易被 AI 抽取的片段是什么
- 哪些标签还不够强
- 哪些问题意图没有覆盖
- 哪些结论缺少证据支撑

推荐模板：

```text
请分析这篇文章的引用潜力。

重点输出：
1. 哪些内容容易被 AI 抽取
2. 哪些证据标签缺失
3. 哪些问答型搜索意图没有覆盖
4. 下一轮最值得增强的 Prompt 约束
```

## 8. 分发 Prompt 应该怎么写

不要直接问：

`这篇文章应该优先发到哪个渠道？`

更好的写法是：

```text
请根据文章主题、读者、证据密度和行业画像，推荐最适合的发布站点。

要求输出：
1. 推荐站点
2. 站点权重理由
3. 适配格式
4. 是否适合自动发布
5. 是否应先发官网或文档站
```

## 9. 创作者如何把 Prompt 做成资产

建议你把 Prompt 分成三层维护。

### 基础层

所有行业共用的规则：

- 结构要求
- 证据类型
- 评分维度
- 返修逻辑

### 行业层

每个行业特有的内容：

- 术语
- 标准体系
- 关键问题
- 禁止误导的结论

### 项目层

每个具体项目或品牌单独维护：

- 目标实体
- 能力边界
- 目标渠道
- 重点关键词

## 10. 自检清单

一个 Prompt 是否合格，可以直接用下面这 10 条检查。

1. 有没有明确读者
2. 有没有明确任务目标
3. 有没有强制结构
4. 有没有证据要求
5. 有没有边界约束
6. 有没有禁用表达
7. 有没有输出格式
8. 能不能被下一步复用
9. 能不能用于评分或返修
10. 能不能沉淀成长期模板

如果有一半答不上来，这个 Prompt 大概率还只是“临时提问”，不是系统资产。
